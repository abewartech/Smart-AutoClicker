# Glossary

-   **Action**: A task performed by the app, such as a Click, Swipe, or Wait.
-   **Condition**: A rule that must be true for a Trigger to fire (e.g., Image Match, Counter Value).
-   **Detection**: The process of finding a Template Image on the current screen.
-   **Event**: A container for Triggers and Actions. "When X happens, do Y."
-   **Flow**: A collection of events that make up a full automation scenario.
-   **Overlay**: The floating UI that draws over other apps, allowing you to control Klick'r.
-   **Template**: A small reference image used to search for buttons/icons on the screen.
-   **Threshold**: The minimum similarity score (0-100%) required to consider an image "found".
-   **Trigger**: The logic that initiates a set of Actions.
